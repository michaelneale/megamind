The problem

You discuss architecture in Claude Code, debug in Codex, prototype in Goose, review in Amp. Each agent starts from zero every time. You end up repeating context, re-explaining decisions, and losing track of what was said where.

megamind is a single binary that searches all their conversation histories in parallel and returns what's relevant. Install it as an agent skill or connect it as an MCP server — pick whichever your tooling supports.

How it works

🔍

Auto-discover

Finds all installed agents by checking their known data paths. No config needed.

⚡

Parallel search

Queries all sources concurrently. Typical search across 7 agents in under 50ms.

🧠

Agent skill

Drop SKILL.md into ~/.skills/ and agents call remember on their own.

🔌

MCP server

Run remember mcp and any MCP client (Claude Desktop, Cursor, etc.) can use it.

🔒

Read-only

Never writes to any agent's database. Just reads local session files.

Install

  1. Clone and build

    git clone https://github.com/michaelneale/megamind.git cd megamind cargo build --release

  2. Put it on your PATH

    cp target/release/remember ~/.local/bin/

    Or wherever you keep binaries.

  3. Connect your agents (pick one)

    Option A: Agent Skill — for agents that support Agent Skills

    mkdir -p ~/.skills/remember && cp SKILL.md ~/.skills/remember/

    Now any agent that reads ~/.skills/ knows how to call remember as a CLI tool.

    Option B: MCP Server — for agents that support MCP (Claude Desktop, Cursor, etc.)

    Add to your MCP client config:

    { "mcpServers": { "remember": { "command": "remember", "args": ["mcp"] } } }

    This exposes a single remember tool over stdio with the same parameters as the CLI (query, keywords, after, before, limit, mode).

  4. Verify

    remember sources

    Should show ✓ next to every agent you have installed.

Usage

$ remember "what was the deploy strategy"
$ remember -k rust -k sqlite
$ remember -k mesh --after 2026-01-01 --before 2026-03-01
$ remember -f json "error handling"
$ remember sources
$ remember clear-cache
$ remember mcp              # start as MCP server over stdio

-k keyword (repeatable) · --after / --before date range · -l limit per source · -f json machine output · --any OR mode
